4. Etiqueta <meta> charset. Volver al Índice ⬆️

<meta charset="UTF-8">

El charset define el juego de caracteres que usará el documento, UTF-8 es la codificación universal para soportar caracteres ajenos al inglés.

Esta etiqueta permite agregar o visualizar caracteres especiales.

UTF-8 tiene 128 caracteres.

Dato importante: Si bien el código ASCII permite desplegar caracteres de forma correcta en el Inglés, no ofrece soporte para otros lenguajes existentes como el Japonés.

Para solucionar este problema existe el código Unicode creado en 1987, y provee soporte para la mayoría de los caracteres existentes en los diferentes lenguajes. La ultima versión soporta 128,000 caracteres. De forma conveniente los primeros 128 caracteres de Unicode son el equivalente al código ASCII. -- Fuente: apuntes.de programador - ASCII Y UTF-8

5. Etiqueta meta viewport. Volver al Índice ⬆️

<meta name="viewport" content="width=device-width, initial-scale=1.0">

la meta etiqueta viewport define como se visualizará el contenido en cualquier dispositivo, sea móvil, tableta, computadora de escritorio. Esta etiqueta es fundamental cuando hacemos diseño responsivo.

Con el atributo name (nombre en español) se indica la zona visible del navegador donde se visualiza o se renderiza el html.

Con el atributo content (contenido en español) se indica el contenido de referencia, con ella podemos adaptar el ancho del documento html al ancho del dispositivo (movil, tablet, laptop, etc).

6. Etiqueta <title> y etiqueta <meta> attribute description. Volver al Índice ⬆️

13. Elementos de linea y de bloque - Tabla periodica de elementos HTML. Volver al Índice ⬆️

13.1. Elementos de linea.

Como su nombre mismo indica, los elementos inline son aquellos cuyo contenido se encuentra en una misma linea, de tal manera que que si colocamos dos o más elementos en linea sin expresar un salto de linea, estos elementos se presentarán uno seguido de otro en una misma linea.

13.2. Elementos de Bloque.

Los elementos de tipo bloque BLOCK son aquellos que se comportan como bloques, cada vez que se presenta un elemento de este tipo se hace un salto de linea de manera predeterminada sin la necesidad de colocar una etiqueta de salto de linea.

13.3. Diferencia entre elementos de linea y de bloque.

la diferencia es que los elementos html de linea no hacen salto de linea sino continúan en una misma linea si no se especifica un salto de linea, en cambio un elemento de tipo bloque empieza y termina en una linea.

16. Etiqueta <img> - Vectores Volver al Índice ⬆️

El elemento de imagen HTML <img> representa una imagen en el documento. Ver: MDN Etiqueta <img>

16.1. Atributos

1 - align.

Alineamiento de la texto respecto al contexto que la rodea.

2 - alt.

Este atributo define el texto alternativo que describe la imagen, texto que los usuarios verán si la URL de la imagen es errónea o la imagen tiene un formato no soportado o si la imagen aún no se ha descargado.

Nota: Omitir este atributo indica que la imagen es una parte clave del contenido, y no tiene equivalencia textual. Establecer este atributo como cadena vacía indica que la imagen no es una parte clave del contenido; los navegadores no gráficos pueden omitirlo.

3 - border.

Anchura del borde alrededor de la imagen.

4 - height.

La altura de la imagen en píxeles CSS en HTML5 o píxeles o como porcentaje en HTML4.

5 - sizes.

Una lista de una o más cadenas separadas por comas indicando el tamaño de la fuente. Cada tamaño de la fuente consiste en:

  1. Codición de medios. Debe omitirse en el último ítem.
  2. Valor del tamaño.El valor del tamaño de la fuente especifica el tamaño de la imagen incrustada.

6 - src

La URL de la imagen. Este atributo es obligatorio para el elemento <img>.

7 - width

El ancho de la imagen en píxeles CSS en HTML5, o píxeles o porcentaje en HTML4.

20. Etiqueta <dl> - lista de definición - definición list. Volver al Índice ⬆️

La etiqueta <dl> en HTML5 se utiliza para crear una lista de definición. Esta lista se compone de una serie de términos y sus definiciones correspondientes.

La etiqueta HTML <dt> se utiliza para definir un término en una lista de definición. Esta etiqueta se utiliza junto con la etiqueta <dl> (lista de definición) y la etiqueta <dd> (definición).

La etiqueta <dt> se coloca antes de la etiqueta <dd> y se utiliza para especificar el término que se está definiendo en la lista. Por ejemplo, en una lista de definición de términos de un glosario, la etiqueta <dt> se utilizaría para definir el término, mientras que la etiqueta <dd> se utilizaría para proporcionar la definición del término.

21. Etiqueta de tabla <table>. Volver al Índice ⬆️

El Elemento de Tabla HTML (<table>) representa datos en dos o mas dimensiones. Ver: MDN - Etiqueta <table>

21.1. Etiquetas HTML dentro de la etiqueta <table>

La estructura básica de una tabla consta de una etiqueta <table> que envuelve a otras etiquetas como <thead>, <tbody>, <tfoot>, <tr>, <th> y <td>. Ver: HTML table | oregoom.

1- Etiqueta <caption>.

1.1.La etiqueta <caption> se usa a menudo con una tabla para describir su contenido. Se utiliza tambien para dar titulos a la tabla. Ver: Etiqueta TABLE | htmldesdecero., htmlquick etiqueta table.

2- Etiqueta <thead> | table head.

2.1. La etiqueta <thead> se usa para crear un encabezado de tabla que se mostrará al principio de la tabla y se utiliza para describir las columnas. Ver: HTML table | oregoom.

2.2. <thead>: representa la sección de encabezado de la tabla. Ver: htmlquick etiqueta table.

3- Etiqueta <tbody>.

3.1. La etiqueta <tbody> se utiliza para crear el cuerpo de la tabla, donde se incluyen las filas y celdas con los datos. Ver: HTML table | oregoom.

1.2 Formatos de audio

Los formatos de audio más comúnmente soportados por los navegadores para la etiqueta <audio> son:

  • MP3 (MPEG-1 Audio Layer III): Es un formato de audio muy popular y ampliamente compatible con la mayoría de los navegadores.
  • OGG (Ogg Vorbis): Es un formato de audio libre y de código abierto, compatible con la mayoría de los navegadores modernos.
  • WAV (Waveform Audio File Format): Es un formato de audio no comprimido, que proporciona una alta calidad de sonido, aunque su tamaño de archivo es relativamente grande.
  • AAC (Advanced Audio Coding): Es un formato de audio desarrollado por MPEG-4 y es ampliamente compatible con los navegadores móviles.

Es importante mencionar que algunos navegadores pueden soportar otros formatos de audio, como FLAC, ALAC, y AIFF, y algunos navegadores no soportan algunos de los formatos mencionados anteriormente, por lo que es recomendable proporcionar varios formatos de audio para garantizar la compatibilidad con la mayoría de los navegadores. 26. Etiqueta <iframe> Volver al Índice ⬆️

Define un contenedor para un contexto de navegación anidado: puede incluir una página web dentro de otra página web. (Traducido con DeepL) Tag iframe - htmlreference.io

La etiqueta <iframe> en HTML se utiliza para insertar contenido de otra página web dentro de una página existente. Es similar a la etiqueta <embed> , pero tiene un mayor soporte en los navegadores y dispositivos. La etiqueta se utiliza dentro de una etiqueta <body> y requiere un atributo “src” que especifica la ubicación de la página web que se va a insertar. Etiqueta <iframe>.

Además, el contenido dentro del iframe es completamente independiente del contenido fuera del iframe, por lo que los estilos CSS y scripts JavaScript no se aplicarán al contenido dentro del iframe a menos que se especifique de forma explícita. Etiqueta <iframe>.

Atributos de Tag iframe

1. src (required)

src: 1. Define la URL de la página web anidada. Puede escribir cualquier URL. Tenga en cuenta que algunos sitios web no permitirán ser iframed. (Traducido con DeepL) Tag iframe - htmlreference.io.

2. Este atributo especifica la ubicación de la página web que se va a insertar en la página actual. Es obligatorio especificar un valor para este atributo. Etiqueta <iframe>.

2. allowfullscreen

allowfullscreen: 1. Permite al navegador mostrar el iframe en pantalla completa con JavaScript. --No se requiere ningún valor.-- (Traducido con DeepL) Tag iframe - htmlreference.io.

2. Define si el contenido del iframe se puede ver a pantalla completa Etiqueta <iframe>.

3. height

Define la altura del iframe en píxeles. (traducido con deepl) Tag iframe - htmlreference.io.

4. width

Define el ancho del iframe en píxeles. (traducido con deepl) Tag iframe - htmlreference.io.

5. name

name: Este atributo especifica el nombre del iframe, el cual se puede utilizar para hacer referencia a él desde otro lugar en la página web. Etiqueta <iframe>.

6. sandobox

sandbox: Este atributo proporciona una serie de restricciones para el contenido dentro del iframe, como no permitir ejecutar scripts, no permitir formularios, etc.

7. seamless

seamless: Este atributo define si el borde y el scrollbar del iframe deben ser ocultos o no.

7. scrolling

scrolling: Este atributo define si se mostrarán barras de desplazamiento en caso de que el contenido sea demasiado grande para el tamaño del iframe. Los valores posibles son “yes”, “no”, “auto”.

8. srcdoc

srcdoc: Este atributo permite especificar el contenido HTML que se mostrará en el iframe, en lugar de cargarlo desde una URL.

9. frameborder

frameborder: Este atributo define si se mostrará un borde alrededor del iframe.
